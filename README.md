# Multitasking — Памятка

---

## Первый запуск (только Discord-администратор сервера)

| Команда | Описание |
|---|---|
| `!роль_админ @роль` | **Первая команда.** Назначить роль, которая управляет ботом |

После этого вся настройка доступна обладателям этой роли.

---

| Команда | Описание |
|---|---|
| `!роль_взп @роль` | Роль ВЗП — тегается в `!взп` |
| `!роль_мп @роль` | Роль МП — тегается в `!взп` и `!мп` |
| `!роль_реаки @роль` | Роль для тега в `!реаки` |
| `!роль_варн 1 @роль` | Роль для варна 1/3 (аналогично 2 и 3) |
| `/тикет_менеджер @роль` | Роль тикет-менеджера — может одобрять/отклонять заявки |
| `/тикет_пинг @роль` | Роль, которая тегается в сообщении тикета |
| `/тикет_доступ @роль` | Добавить роль с доступом к просмотру тикетов (можно несколько) |
| `/тикет_доступ_убрать @роль` | Убрать роль из доступа к тикетам |
| `/тикет #канал Категория` | Создать панель заявок в канале |
| `/лог_отказов #канал` | Канал для дублирования отказов по заявкам |
| `/магазин` | Развернуть панель магазина в текущем канале |
| `!афк` | Создать панель АФК в текущем канале |
| `!инактив` | Создать панель инактива в текущем канале |
| `!контракт` | Создать панель контрактов в текущем канале |
| `!feedback` | Создать панель обратной связи в текущем канале |
| `/настройки` | Показать текущую конфигурацию бота (роли, каналы, панели) |

## Сборы (только Администратор)

| Команда | Описание |
|---|---|
| `!взп [кол-во] [название]` | Сбор ВЗП. Тегает **роль ВЗП + роль МП** |
| `!мп [кол-во] [название]` | Сбор МП. Тегает **роль МП** |
| `!реаки [кол-во] [название]` | Сбор мероприятия. Тегает **роль реаки** |

- Можно прикрепить фото к сообщению — оно появится в эмбеде
- До 24 слотов — кнопки с номерами, 25+ — одна кнопка ✅
- Создаётся тред с живым списком участников

| Команда | Где | Описание |
|---|---|---|
| `!замена <айди_кого> <айди_на_кого>` | В треде сбора | Заменить участника в слоте (0 = убрать) |

## Баллы и варны (только Администратор)

| Команда | Описание |
|---|---|
| `!дать @пользователь 500` | Начислить баллы |
| `!снять @пользователь 500` | Снять баллы |
| `!баланс [@пользователь]` | Показать баллы и варны |
| `!warn @пользователь 2 причина` | Выдать варн (1/2/3) — назначает соответствующую роль + ЛС |
| `!снять_варн @пользователь` | Снять варн и убрать варн-роль |
| `!warnlist` | Список всех участников с варнами |

## Магазин (только роль администратора)

| Команда | Описание |
|---|---|
| `/магазин` | Развернуть панель магазина в канале |
| `/добавить_товар` | Добавить товар (открывает форму) |
| `/убрать_товар <id>` | Удалить товар по ID |
| `/список_товаров` | Список всех товаров с ID |
| `/товар_роль <id> @роль` | Привязать роль к товару типа `give_role` |
| `/лог_магазина #канал` | Канал, куда пишутся все покупки из магазина |
| `/роль_магазина @роль` | Роль, которая тегается при покупках с ручной выдачей (`notify`) |

**Типы действий товара:**
| Тип | Что происходит |
|---|---|
| `remove_warn` | Автоматически снимает варн покупателю |
| `give_role` | Автоматически выдаёт привязанную роль |
| `notify` | Списывает баллы + уведомление в лог-канал (ручная выдача) |

## Тикеты (заявки)

**Для участников:**
- Нажать кнопку **📋 Подать заявку** в панели
- Заполнить форму: ник/статик, часы, возраст, семьи, откат
- После создания тикета можно писать в канале

**Для тикет-менеджера:**
- ✅ **Одобрить** — участнику приходит ЛС с одобрением, канал удаляется через 10 сек
- ❌ **Отклонить** — указать причину → ЛС участнику + запись в лог-канал

**Лог отказа содержит:** участника, причину, кто рассматривал

## Приватные комнаты

| Команда | Описание |
|---|---|
| `/приват канал_создания категория канал_панели` | Настроить систему приватных комнат |

**Как работает:**
1. Администратор создаёт голосовой канал-триггер и настраивает `/приват`
2. Участник заходит в канал-триггер → бот создаёт персональный голосовой канал и присылает панель управления

**Кнопки панели:**
| Кнопка | Действие |
|---|---|
| ➕ / ➖ | Добавить / убрать 1 слот |
| 👥 | Задать точное количество слотов |
| 🔓 / 🔒 | Открыть / закрыть канал для всех |
| 👤 / 🚷 | Добавить / убрать конкретного пользователя |
| 🙈 / 👁 | Скрыть / показать канал |
| 🔄 | Передать канал другому пользователю |
| ✏️ | Переименовать канал |
| 🚫 | Заблокировать пользователя (не сможет зайти) |

Когда все выходят из канала — он удаляется автоматически вместе с панелью.

## Обратная связь (Фидбеки)

**Настройка (только Администратор):**

| Команда | Описание |
|---|---|
| `!feedback` | Создать панель обратной связи в текущем канале |
| `!feedback_канал #канал` | Канал, куда будут приходить фидбеки (для старшего состава) |
| `!feedback_роль @роль` | Роль, которая тегается при каждом новом фидбеке |
| `!feedback_текст <текст>` | Изменить текст панели (сразу обновится в канале) |
| `!feedback_фото <url>` | Изменить фото панели (сразу обновится в канале) |

**Как работает:**

1. Администратор прописывает `!feedback` в канале для участников — появляется embed с кнопкой **📝 Оставить предложение**
2. Участник нажимает кнопку → открывается форма с одним полем "Ваше предложение"
3. После отправки в канал старшего состава приходит embed с:
   - Аватаром и именем отправителя
   - Тегом настроенной роли
   - Текстом предложения
4. Под сообщением **автоматически создаётся тред** для обсуждения

## Контракты

**Настройка (только Администратор):**

| Команда | Описание |
|---|---|
| `!контракт` | Создать/обновить панель контрактов в текущем канале |
| `/контракт_текст <текст>` | Изменить текст панели контрактов |
| `/контракт_фото <url>` | Изменить фото панели контрактов |
| `/контракт_роль @роль` | Роль, которая тегается при создании каждого контракта |

**Как работает:**

1. Администратор прописывает `!контракт` в нужном канале — появляется панель с кнопкой **✅ Взять контракт**
2. Участник нажимает кнопку → открывается форма:
   - **Длительность** — например `2:20`
   - **Начало** — например `10:20 / сейчас / через 15 минут`
3. После отправки формы в канал отправляется новый embed с тегом роли и тремя кнопками:

| Кнопка | Кто может | Действие |
|---|---|---|
| ➕ Принять участие | Любой | Вписывается в список участников |
| ➖ Отменить участие | Любой | Выписывается из участников |
| 🔒 Контракт взят | Создатель или Администратор | Удаляет embed контракта |

## Статистика GTA5RP

| Команда | Описание |
|---|---|
| `/статистика` | Показать онлайн всех серверов GTA5RP в текущем канале |

- Эмбед автоматически обновляется каждые **30 секунд**
- Серверы отсортированы в фиксированном порядке: Downtown, Strawberry, VineWood, Blackberry, Insquad, Sunrise, Rainbow, Richman, Eclipse, La Mesa, Burton, Rockford, Alta, Del Perro, Davis, Harmony, Redwood, Hawick, Grapeseed, Murrieta, Vespucci, Milton, La Puerta
- Внизу показывается общий онлайн по проекту
- Данные берутся из публичного RageMP API

## Сохранение данных

Все данные автоматически сохраняются в файл **`data.json`** рядом с `main.py` и восстанавливаются при перезапуске бота.

**Что сохраняется:**

| Данные | Описание |
|---|---|
| Баллы участников | Валюта для магазина |
| Варны | Количество, причина, кто выдал |
| АФК-список | Кто в АФК, причина, время возврата |
| Инактив-список | Кто в инактиве, причина, дата возврата |
| Сборы и слоты | Все активные сборы с занятыми слотами |
| Панели | АФК, инактив, магазин (ID каналов и сообщений) |
| Товары магазина | Все добавленные товары |
| Все роли | Администратор, МП, ВЗП, варн-роли, тикет-роли и др. |
| Настройки сборов | Роли доступа к `!взп`, `!мп`, `!реаки` |

> Файл `data.json` нельзя удалять — в нём хранится всё состояние бота.
